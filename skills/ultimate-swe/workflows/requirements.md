# 1. Requirements Gathering

## Objective

Fully understand the problem before any design or implementation begins.

---

## Key Activities

- Identify business goals
- Define functional requirements
- Define non-functional requirements
- Clarify user experience expectations
- Identify constraints (technical, legal, performance)
- Define acceptance criteria
- Remove ambiguity through questions

---

## Output

- Clear requirements document
- Measurable acceptance criteria
- Confirmed scope boundaries

---

## Rule

Never assume requirements. If unclear, ask questions before proceeding.
